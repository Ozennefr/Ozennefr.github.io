A genetic algorithm in Julia

Why Julia

Taken directly for Julialang official website : "Julia is a high-level, high-performance dynamic programming language for numerical computing. It provides a sophisticated compiler, distributed parallel execution, numerical accuracy, and an extensive mathematical function library. Julia’s Base library, largely written in Julia itself, also integrates mature, best-of-breed open source C and Fortran libraries for linear algebra, random number generation, signal processing, and string processing. In addition, the Julia developer community is contributing a number of external packages through Julia’s built-in package manager at a rapid pace. IJulia, a collaboration between the Jupyter and Julia communities, provides a powerful browser-based graphical notebook interface to Julia."

Here is a list of some of the reasons why I chose Julia :

  • Faster than pyhon (nearly as fast as C in some cases)
  • Types and abstract types
  • fresh and usefull conventions (such as the "!" as a suffix, indicating a function modifies its argument(s)).
  • Meta programming capabilities
  • Macros
  • Mocha, a julia implementation of the Cofee C++ deep learning framework (although it is not used here, it's a good enough reason to investigate the Julia language)
  • Python friendly syntaxe, making the switch from python to julia easy

This new language also comes with its disavantages, namely :

  • Lack of documentation regarding Plotting

  • A bubbling package ecosystem with a lots of diversity and activity, which leads to the lack of standards.

  • Sometimes obscure debug messages

Genetic algorithms

Genetic algorithms are a special class of optimisation methods, inspired by biology.

The general skeleton of a GA is the following :

  • Generate a population of potential solutions
  • Until we reach the desired precision / generation
  • for every individual
    • decode the chromosome
    • evaluate the individual
  • Until we have a big enough new generation
    • select two individuals
    • create two offspring from them
    • mutate the offsprings

Each individual is represented by a chromosome, which encode the information necessary to evaluate him through the fitness function (the function to optimize).

Custom types

As established earlier, the type system of Julia is really powerful. Here, we use it in place of more traditionnal objects like dicts or structures.

type Problem
    fitness #fonction de n variables
    numberOfVar::Int64
    range::Array{Float64,1}
end

type Individual
    id::Int64
    generation::Int64
    genotype::Array{Int64,1}
    phenotype::Array{Float64,1}
    fitness::Float64
end

Base methods override

Using custom types allow us to override base methods fo our use.

#crossover = ind*ind
function Base.:*(pere::Individual, mere::Individual)
    #1 points crossover
    point = rand(1:(length(pere.genotype)))
    chr1 = [pere.genotype[1:point]..., mere.genotype[1+point:end]...]
    chr2 = [mere.genotype[1:point]..., pere.genotype[1+point:end]...]
    offspring1 = Individual(0, 0, chr1, [], Inf)
    offspring2 = Individual(0, 0, chr2, [], Inf)
    return(offspring1, offspring2)
end

The first genetic operator, the crossover, is simply implemented as what's known as a "one point crossover".

Building blocks

The population is initialized using the Individual reator function. Fitness is initialized at $\infty$ as we want to minimize it.

function initPopulation(popSize::Int64, chromosomeSize::Int64)
    population = Array{Individual}(popSize);
    for iInd in 1:popSize
        population[iInd] = Individual(iInd,1,rand(Bool, (chromosomeSize)),[],Inf);
    end
    return(population)
end;

The decoding function is the most complicated part of the algorithm. The chromosomes we use are binary encoded, we firstly decode them as an unsigned integer, and then restrict the value to a float in the range we want.

function expressGenotype!(individual::Individual, problem::Problem)
    genePerVar = round(Int64,(size(individual.genotype)[1]/problem.numberOfVar))
    individual.phenotype = Array{Float64}(problem.numberOfVar)

    for iVar in 1:problem.numberOfVar
        individual.phenotype[iVar] = 0;
        for iGene in 1:genePerVar
            individual.phenotype[iVar] += individual.genotype[iGene+(iVar-1)*genePerVar]*2.0^(iGene-1)
        end
        individual.phenotype[iVar] = -problem.range[iVar] + 2.0*problem.range[iVar]/(1.-2.0.^(-genePerVar)) .*
                                    2.0^(-genePerVar).*individual.phenotype[iVar]
    end
end

Because our fitness function is stored as a Julia function, it is trivial to evaluate an individual.

function evaluateIndividual!(individual::Individual, problem::Problem)
    individual.fitness = problem.fitness(individual.phenotype...);
end

The selection operation is implemented as a tournament selection. In this case, we select a few individuals (tournament size) at random in the population. Then, the tournament takes place : with the probability ptour, the best individual from the sampled population is selected. otherwise, we discard it and continue the tournament with the remaining individuals. 

function selectIndividual(tournamentSize::Int64, ptour::Float64, pop::Array{Individual,1})
    selected = Individual(0, 0, [], [], Inf)
    fitnesses = Array{Float64}(tournamentSize)
    subPop = sample(pop, tournamentSize, replace=true)
    for j in 1:tournamentSize
        fitnesses[j] = subPop[j].fitness
    end
    notSelected = true
    while notSelected
        mxval, mxindx = findmin(fitnesses)
        if rand(Float64) < ptour
            notSelected = false
            selected = subPop[mxindx]
        elseif length(fitnesses)==1
            selected = subPop[mxindx]
        else
            deleteat!(fitnesses, mxindx)
        end
    end
    return(selected)
end

When we want to select multiple individuals, we simply loop until the desired amount is samled.

function selectIndividuals(nInd::Int64, tournamentSize::Int64, ptour::Float64, pop::Array{Individual,1})
    selected = Array{Individual,1}(nInd)
    for i in 1:nInd
        selected[i] = selectIndividual(tournamentSize, ptour, pop)
    end
    return(selected)
end

With our binary representation, the mutation operator becomes trivial. It is simply implemented as a random bitflip somewhere on the chromosome.

function mutate!(ind::Individual, pmut::Float64)
    for i in 1:length(ind.genotype)
        if rand(Float64) < pmut
            ind.genotype[i] = (ind.genotype[i]+1)%2 #bitflip
        end
    end
end;

Plotting progress

To make sure our algorithm is running smoothly, and to monitor the convergence, we need to plot some statistics helping us tweak the numerous parameters.

function plotPopulation(pop::Array{Individual,1}, problem::Problem, fitnessHistory)

    nVar = problem.numberOfVar
    nIndiv = length(pop)

    ph = Array{Float64,2}(nIndiv, nVar)
    f = Array{Float64,1}(nIndiv)
    g = Array{Int64,1}(nIndiv)
    for i in 1:nIndiv
        ph[i,:] = pop[i].phenotype
        f[i] = pop[i].fitness
        g[i] = pop[i].generation
    end

    y = [histogram(ph[:,i], title="y$i", nbins = 20) for i in 1:nVar]
    y_plot = plot(y..., layout=(1,nVar))

    fp = histogram(f, title="fitness distribution", nbins = 20)
    gp = histogram(g, title="generation distribution", nbins = 20)
    statp = plot(fp, gp, layout = (2,1))
    fh = plot(fitnessHistory, title="fitness history")
    fplot = plot(fh, statp, layout=(1,2))
    plot(y_plot, fplot, layout=(2,1))
    #scatter(ph[:,1],ph[:,2],zcolor=f)
    IJulia.clear_output(true)
    plot!(legend=false) |> display
end

The next generation

From all of the functions we already made, this function is pretty straightforward to implement.

function makeNewGeneration(pop::Array{Individual,1}, gen::Int64, ptour::Float64, 
                            tournamentSize::Int64, carryOver::Float64, pmut::Float64,
                            elitism::Bool, bestInd::Int64, pcross::Float64)
    popSize = length(pop)
    newGen = Array{Individual}(popSize)
    if carryOver != 0.
        newGen[1:round(Int, carryOver*popSize)] = selectIndividuals(round(Int, carryOver*popSize), tournamentSize, ptour, pop)
    end
    for i in round(Int, carryOver*popSize)+1:2:popSize
        if rand(Float64) < pcross
            offspring1, offspring2  = *(selectIndividuals(2, tournamentSize, ptour, pop)...)
        else
            parents = selectIndividuals(2, tournamentSize, ptour, pop)
            offspring1, offspring2 = (parents[1], parents[2])
        end
        offspring1.id = i+gen*popSize
        offspring1.generation = 1+gen
        offspring2.id = i+1+gen*popSize
        offspring2.generation = 1+gen
        mutate!(offspring1, pmut)
        mutate!(offspring2, pmut)
        newGen[i] = offspring1
        newGen[i+1] = offspring2
    end
    if elitism
        newGen[1]=pop[bestInd]
    end
    return(newGen)
end
;

Genetic algorithm

Here is the final function, wich encaplsulate all of the steps above.

function geneticOptimisation(problem::Problem; chromosomeSize::Int64=128, 
                            popSize::Int64=10000, years::Int64=100, 
                            ptour::Float64=0.8, tournamentSize::Int64=5,
                            carryOver::Float64 = 0., pmut::Float64 = 0.1,
                            elitism::Bool = false, pcross::Float64 = 0.8)
    pop = initPopulation(popSize, chromosomeSize)
    fitnessHistory = Array{Float64,1}(years)
    bestFitness = Inf
    bestIndiv = 0
    fill!(fitnessHistory, 0)
    for gen in 1:years
        bestFitness = Inf
        bestIndiv = 0
        for i in 1:popSize
            expressGenotype!(pop[i], problem);
            evaluateIndividual!(pop[i], problem);
            if pop[i].fitness < bestFitness
                bestFitness = pop[i].fitness
                bestIndiv = i
            end
        end
        fitnessHistory[gen] = bestFitness
        if gen%100 == 1
            plotPopulation(pop, problem, fitnessHistory)
        end
        if gen <years
            pop = makeNewGeneration(pop, gen, ptour, tournamentSize, carryOver, pmut, elitism, bestIndiv, pcross)
        end
    end
    return(pop, bestFitness, bestIndiv)
end
;

We can see that we obtain a pretty good estimate of the minimum. The minimum fitness varies from generation to generation, displaying the randomness of the algorithm. Enabling elitism and carryover can help smoothing out these iregularities.

What's next

Obviously, this simple problem isn't really convincing. Genetic algorithms are a fun class of optimisation methods that really shines in two situations : - problems were the gradient is not defined (heterogeneous variable types, reinforcement learning) - problems with a lot of local minima.

Here the aim was simply to have fun implementing this weird yet powerfull algorithm !
